# AI Project 技术文档

## 1. 项目概述

AI Project 是一个智能监控系统，集成了环境数据采集、设备管理、机器人控制、AI智能分析等功能。项目采用Express框架构建，使用MySQL作为数据库，集成了Qwen大语言模型实现智能化功能增强。

### 1.1 核心功能

- **用户认证与管理**：JWT认证、用户CRUD操作
- **设备状态监控**：设备统计、状态监控
- **传感器管理**：传感器CRUD、数据采集
- **环境数据监控**：温度、湿度、烟雾浓度监测
- **系统状态监控**：数据库连接、内存使用、CPU负载、磁盘使用、网络状态
- **AI智能分析**：环境预测、故障预测、异常检测、智能报告
- **自然语言交互**：智能问答、聊天对话

### 1.2 技术栈

| 技术           | 版本       | 用途          |
| ------------ | -------- | ----------- |
| Node.js      | 14.x+    | 运行环境        |
| Express      | \~4.16.1 | Web框架       |
| MySQL        | 5.7+     | 数据库         |
| mysql2       | ^3.20.0  | MySQL驱动     |
| jsonwebtoken | ^9.0.3   | JWT认证       |
| bcrypt       | ^6.0.0   | 密码加密        |
| socket.io    | ^4.8.3   | WebSocket支持 |
| axios        | ^1.13.6  | HTTP客户端     |
| winston      | ^3.11.0  | 日志管理        |
| Qwen AI      | -        | 大语言模型       |

- **请求参数**:

| 参数名      | 类型     | 必填 | 描述  |
| -------- | ------ | -- | --- |
| username | string | 是  | 用户名 |
| password | string | 是  | 密码  |
| phone    | string | 是  | 手机号 |

- **请求示例**:

```json
{
  "username": "admin",
  "password": "123456",
  "phone": "13800138000"
}
```

- **响应示例**:

```json
{
  "code": 200,
  "message": "登录成功",
```

### 6.2 传感器数据表结构

```sql
CREATE TABLE sensor_data (
  id INT AUTO_INCREMENT PRIMARY KEY,
  device_id VARCHAR(50),
  temperature DECIMAL(5,2),
  humidity DECIMAL(5,2),
  smoke_level INT,
  max_temp DECIMAL(5,2),
  human_detected BOOLEAN DEFAULT FALSE,
  fire_risk VARCHAR(20) DEFAULT 'low',
  env_status VARCHAR(20) DEFAULT 'normal',
  battery INT DEFAULT 100,
  record_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_device_id (device_id),
  INDEX idx_record_time (record_time)
) PARTITION BY RANGE (TO_DAYS(record_time)) (
  PARTITION p202604 VALUES LESS THAN (TO_DAYS('2026-05-01')),
  PARTITION p202605 VALUES LESS THAN (TO_DAYS('2026-06-01')),
  PARTITION p202606 VALUES LESS THAN (TO_DAYS('2026-07-01')),
  PARTITION p202607 VALUES LESS THAN (TO_DAYS('2026-08-01')),
  PARTITION p202608 VALUES LESS THAN (TO_DAYS('2026-09-01')),
  PARTITION p202609 VALUES LESS THAN (TO_DAYS('2026-10-01')),
  PARTITION p202610 VALUES LESS THAN (TO_DAYS('2026-11-01')),
  PARTITION p202611 VALUES LESS THAN (TO_DAYS('2026-12-01')),
  PARTITION p202612 VALUES LESS THAN (TO_DAYS('2027-01-01')),
  PARTITION pmax VALUES LESS THAN MAXVALUE
);
```

### 6.3 用户表结构

```sql
CREATE TABLE users (
  id VARCHAR(50) PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  email VARCHAR(100) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  phone VARCHAR(20),
  role_id INT DEFAULT 2,
  status ENUM('active', 'inactive') DEFAULT 'inactive',
  last_login DATETIME,
  failed_login_attempts INT DEFAULT 0,
  last_failed_login DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_username (username),
  INDEX idx_email (email),
  INDEX idx_status (status)
);
```

AI Project 技术文档

1. 项目概述

AI Project 是一个智能监控系统，集成了环境数据采集、设备管理、机器人控制、AI智能分析等功能。项目采用Express框架构建，使用MySQL作为数据库，集成了Qwen大语言模型实现智能化功能增强。

1.1 核心功能

  • 用户认证与管理：JWT认证、用户CRUD操作
  • 设备状态监控：设备统计、状态监控
  • 传感器管理：传感器CRUD、数据采集
  • 环境数据监控：温度、湿度、烟雾浓度监测
  • 系统状态监控：数据库连接、内存使用、CPU负载、磁盘使用、网络状态
  • AI智能分析：环境预测、故障预测、异常检测、智能报告
  • 自然语言交互：智能问答、聊天对话

4. API接口文档

4.1 认证接口

4.1.1 用户登录

  • URL: POST /api/auth/login
  • 描述: 用户登录获取JWT token
  • 请求参数:
参数名 类型 必填 描述
username string 是 用户名
password string 是 密码
phone string 是 手机号
  • 请求示例:
{"username": "admin", "password": "123456", "phone": "13800138000"}
  • 响应示例:
{"code": 200, "message": "登录成功", "data": {"token": "eyJhbGciOiJIUzI1NiIs...", "userInfo": {"id": "1", "username": "admin", "role": "1", "avatar": null}}, "timestamp": "2026-04-02T00:00:00Z"}

4.1.2 用户注册

  • URL: POST /api/auth/register
  • 描述: 用户注册
  • 请求参数:
参数名 类型 必填 描述
username string 是 用户名
password string 是 密码
confirmPassword string 是 确认密码
email string 是 邮箱
phone string 是 手机号

4.6 AI功能接口

4.6.1 环境数据预测

  • URL: POST /api/ai/predict/environment
  • 描述: 基于历史环境数据预测未来趋势
  • 请求参数:
参数名 类型 必填 描述
data array 是 历史环境数据数组
  • 请求示例:
{"data": [{"timestamp": "2026-04-01T00:00:00Z", "temperature": 25, "humidity": 60, "pm25": 15}, {"timestamp": "2026-04-01T01:00:00Z", "temperature": 24, "humidity": 62, "pm25": 18}]}
  • 响应示例:
{"code": 200, "message": "success", "data": {"prediction": "基于历史数据分析，未来24小时温度将保持在23-26°C之间..."}, "timestamp": "2026-04-02T00:00:00Z"}

4.6.2 设备故障预测

  • URL: POST /api/ai/predict/device-failure
  • 描述: 预测设备故障风险
  • 请求参数:
参数名 类型 必填 描述

  • 请求示例:
{"deviceData": {"deviceId": "D001", "status": "online", "temperature": 45, "operatingHours": 1200}}

4.6.3 异常检测

  • URL: POST /api/ai/detect/anomalies
  • 描述: 检测数据中的异常模式
  • 请求参数:
参数名 类型 必填 描述
data array 是 待检测数据
type string 是 数据类型

4.6.4 智能分析报告

  • URL: POST /api/ai/generate/report
  • 描述: 生成智能分析报告
  • 请求参数:
参数名 类型 必填 描述
data array 是 分析数据
reportType string 是 报告类型

4.6.5 自然语言查询

  • URL: POST /api/ai/query
  • 描述: 处理自然语言查询
  • 请求参数:
参数名 类型 必填 描述
query string 是 查询问题
context object 否 上下文信息
  • 请求示例:
{"query": "今天的环境数据如何？", "context": {"type": "environment", "date": "2026-04-02"}}
  • 响应示例:
{"code": 200, "message": "success", "data": {"response": "根据今天的环境数据，温度保持在25°C左右，湿度60%，空气质量良好..."}, "timestamp": "2026-04-02T00:00:00Z"}

  • URL: GET /api/ai/analyze/environment

CREATE TABLE sensor_data (
  id INT AUTO_INCREMENT PRIMARY KEY,
  device_id VARCHAR(50),
  temperature DECIMAL(5,2),
  humidity DECIMAL(5,2),
  smoke_level INT,
  max_temp DECIMAL(5,2),
  human_detected BOOLEAN DEFAULT FALSE,
  fire_risk VARCHAR(20) DEFAULT 'low',
  env_status VARCHAR(20) DEFAULT 'normal',
  battery INT DEFAULT 100,
  record_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_device_id (device_id),
  INDEX idx_record_time (record_time)
) PARTITION BY RANGE (TO_DAYS(record_time)) (
  PARTITION p202604 VALUES LESS THAN (TO_DAYS('2026-05-01')),
  PARTITION p202605 VALUES LESS THAN (TO_DAYS('2026-06-01')),
  PARTITION p202606 VALUES LESS THAN (TO_DAYS('2026-07-01')),
  PARTITION p202607 VALUES LESS THAN (TO_DAYS('2026-08-01')),
  PARTITION p202608 VALUES LESS THAN (TO_DAYS('2026-09-01')),
  PARTITION p202609 VALUES LESS THAN (TO_DAYS('2026-10-01')),
  PARTITION p202610 VALUES LESS THAN (TO_DAYS('2026-11-01')),

6.3 用户表结构

CREATE TABLE users (
  id VARCHAR(50) PRIMARY KEY,

9. 总结

AI Project 通过集成Qwen大语言模型，实现了从传统监控系统到智能监控系统的升级。AI功能的加入使得系统能够：

  • 预测环境趋势：提前预警，优化资源配置

  • 检测数据异常：实时监控，及时发现问题
  • 生成智能报告：提供决策支持
  • 自然语言交互：提高系统易用性

项目具备良好的扩展性和维护性，可以根据业务需求持续添加新的AI功能模块。
